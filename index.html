AI開発研修ポータル

GitHub、Codex、Claude Codeを使って、開発の流れを少しずつ学ぶための入口です。

まずは全体像をつかむ

このポータルでは、資料を読むだけでなく、実際に手を動かしながら学びます。

Step 1

GitHubに記録する

README、要件定義書、学習ログ、プロンプト集を保存して、あとから見返せる状態にします。

Step 2

Codexで作る

画面や機能をCodexに依頼し、小さく動くアプリを作っていきます。

Step 3

Claude Codeで見直す

作ったコードを読みやすくしたり、改善点を確認したりします。

Step 4

用語を覚えて確認する

まず用語学習で意味をつかみ、そのあと用語クイズで理解度を確認します。

実践課題

完了条件を見ながら、ひとつずつ進めます。

用語学習

クイズの前に、システム開発でよく使う言葉をやさしく確認します。

問題文

GitHubガイド

今日使った操作を、初心者向けに整理しています。

GitHubとは

開発で使うファイルや資料を保存・共有するためのサービスです。

リポジトリ

プロジェクトを入れる箱です。今回の例では ai-dev-training-portal がリポジトリです。

README

プロジェクトの説明書です。リポジトリを開いた人が最初に読む場所です。

Commit

変更を保存することです。ファイルを作ったり直したりしたらCommitします。

新しいファイルを作る手順

  1. リポジトリを開く
  2. Add file を押す
  3. Create new file を押す
  4. ファイル名と本文を入力する
  5. Commit changes で保存する

プロンプト集

よく使う依頼文をカテゴリ別に残して、すぐ取り出せるようにします。

プロンプト記録テンプレート

目的、入力した文章、結果、改善点をセットで残します。

# プロンプト記録

## 使ったツール
- ChatGPT
- Codex
- Claude Code

## 目的
何をしたくて使ったかを書く。

## 入力したプロンプト
ここに実際の依頼文を書く。

## 返ってきた結果
回答や作られたものを書く。

## 次に試すこと
改善したい聞き方を書く。
