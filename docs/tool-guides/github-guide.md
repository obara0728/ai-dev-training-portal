# GitHub初心者ガイド

## GitHubとは

GitHubは、開発で使うファイルや資料を保存・共有するためのサービスです。

この研修では、以下の目的で使います。

- 作ったアプリを保存する
- 要件定義書や学習ログを保存する
- 作業の履歴を残す
- 社内メンバーに共有する

## よく使う言葉

### リポジトリ

プロジェクトを入れる箱です。

例: `ai-dev-training-portal`

### README

プロジェクトの説明書です。

GitHubでリポジトリを開いたときに最初に見える説明文です。

### docsフォルダ

資料を入れる場所です。

この研修では、要件定義書、学習ログ、プロンプト集、課題一覧などを入れます。

### Commit

変更を保存することです。

ファイルを作ったり直したりしたら、CommitしてGitHubに記録します。

## 基本操作

## 1. 新しいファイルを作る

1. リポジトリを開く
2. `Add file` を押す
3. `Create new file` を押す
4. ファイル名を入力する
5. 本文を書く
6. `Commit changes...` を押す
7. もう一度 `Commit changes` を押す

## 2. フォルダを作る

GitHubでは、フォルダだけを単独で作ることはできません。

次のように、ファイル名に `/` を入れるとフォルダも一緒に作られます。
